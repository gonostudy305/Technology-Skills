📝 Checklist Sẵn sàng

    ✨ Mô phỏng Xử lý Sự cố với Gemini

    Nhập một tình huống rủi ro, AI sẽ lập kịch bản phối hợp cho đội ngũ của bạn.

    🎯 Từ bài học này ra thực tế — 3 bước tiếp theo

    1. Xây Release Checklist cho team

    Lấy checklist của 4 vai trò (PM, Dev, QA, DevOps) trong bài, customize theo project môn học của bạn. Chia checklist thành 3 giai đoạn: Pre-release, Release day, Post-release.

    2. Tập mô phỏng sự cố

    Hẹn nhóm 30 phút, đặt ra 1 tình huống (VD: "database migration lỗi 10 phút trước release"). Mỗi người đóng 1 vai trò và đề xuất hành động trong 5 phút.

    3. Tìm hiểu CI/CD pipeline

    Tạo một GitHub Actions workflow đơn giản: push code → chạy test → build → deploy lên Vercel/Firebase Hosting. Đây là kỹ năng DevOps nền tảng.

    📝 Ghi chú về triển khai trang này

    Trang này sử dụng Firebase Auth (anonymous sign-in) + Firestore (lưu custom roles) + Gemini API (mô phỏng sự cố). Khi chạy standalone (ngoài Google Canvas), bạn cần: (1) tạo Firebase project và thay __firebase_config bằng config thật, (2) bật Anonymous Auth trong Firebase Console, (3) không nhúng Gemini API key trực tiếp vào client-side — thay vào đó, tạo một proxy endpoint (ví dụ: Cloud Function hoặc Next.js API route) nhận prompt từ client rồi gọi Gemini API ở server-side để tránh lộ key.
