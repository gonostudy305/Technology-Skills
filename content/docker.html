TL;DR: Docker giúp đóng gói ứng dụng thành image bất biến để chạy nhất quán từ máy dev đến production. Để vận hành ổn định, bạn cần chuẩn Dockerfile, phân tách config bằng biến môi trường, quản lý state qua volume và theo dõi container bằng healthcheck + logs tập trung.

1. Docker là gì và vì sao cần dùng?

Docker là nền tảng containerization cho phép bạn đóng gói source code, runtime, dependency và cấu hình vào một image duy nhất.

Lợi ích chính

  • Triển khai nhất quán giữa dev, staging và production.
  • Khởi động nhanh, tiết kiệm tài nguyên hơn VM truyền thống.
  • Dễ scale theo dịch vụ với Compose hoặc Kubernetes.

2. Cấu trúc Dockerfile chuẩn

Nguyên tắc

  • Dùng base image nhỏ, rõ version.
  • Tận dụng layer cache (COPY package trước source).
  • Không chạy bằng root trong production.
FROM node:20-alpine WORKDIR /app COPY package*.json ./ RUN npm ci --omit=dev COPY . . ENV NODE_ENV=production EXPOSE 3000 CMD ["npm", "run", "start"]

3. Runtime: Container, Network, Volume

Container

Chỉ nên chạy một tiến trình chính cho mỗi container để dễ quan sát và scale.

Network

Đặt service vào network riêng, chỉ expose cổng public thật sự cần thiết.

Volume

Dữ liệu trạng thái (database, uploads, logs) phải nằm ngoài container để tránh mất dữ liệu khi recreate.

4. Docker Compose cho môi trường nhiều dịch vụ

Compose giúp khai báo stack theo mã, chạy đồng thời API, database, cache, queue với một lệnh duy nhất.

services: api: build: . ports: - "3000:3000" env_file: - .env depends_on: - db db: image: postgres:16-alpine volumes: - pgdata:/var/lib/postgresql/data volumes: pgdata:
Lưu ý: `depends_on` chỉ đảm bảo thứ tự khởi chạy, không đảm bảo dịch vụ đã sẵn sàng. Hãy bổ sung healthcheck hoặc retry logic ở ứng dụng.

5. Bảng lỗi phổ biến và cách xử lý

Triệu chứng Nguyên nhân thường gặp Cách xử lý
Container restart liên tục CMD sai, thiếu env, app crash khi boot Kiểm tra logs, chạy lại bằng shell, thêm healthcheck.
Kết nối DB thất bại Network sai, DB chưa sẵn sàng, sai host Dùng tên service Compose làm host, thêm retry khi startup.
Image build quá chậm Dockerfile không tối ưu cache layer Copy lockfile trước, dùng multi-stage build.
Deploy xong mất dữ liệu Không mount volume persistent Tách state ra volume hoặc managed storage.

6. Checklist production

  1. Pin version image và dependency, tránh latest.
  2. Bật healthcheck, log tập trung và alert cơ bản.
  3. Không hard-code secret trong image.
  4. Bổ sung resource limits (CPU/RAM) cho từng service.
  5. Tự động scan lỗ hổng image trước khi phát hành.

7. FAQ nhanh

Khi nào dùng Docker, khi nào dùng VM?

Docker phù hợp ứng dụng cloud-native và microservices; VM phù hợp workload cần kernel riêng hoặc cô lập mức hệ điều hành.

Docker có thay thế hoàn toàn Kubernetes không?

Không. Docker là nền tảng build/run container, Kubernetes là hệ điều phối container ở quy mô lớn.

Compose có dùng được cho production không?

Có thể ở quy mô nhỏ, nhưng hệ lớn nên dùng orchestrator chuyên dụng như Kubernetes hoặc Nomad.
